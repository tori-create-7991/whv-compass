# 1. Record architecture decisions

Date: 2026-06-23

## Status

Accepted

## Context

This is a public OSS project; contributors need to understand *why* the
architecture is the way it is, not just *what* it is.

## Decision

We use lightweight Architecture Decision Records (ADRs), one Markdown file per
decision in `docs/adr/`, numbered sequentially. Format: Status, Context,
Decision, Consequences. Superseded ADRs stay in the repo and link to their
replacement.

## Consequences

- Decisions are discoverable and reviewable in PRs.
- New contributors can read the ADR trail instead of asking.
